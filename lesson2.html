01

舊方法的瓶頸：循序閱讀

在 Transformer 出現之前，語言模型（RNN）是一個字接一個字、照順序閱讀的。 問題在於：讀到句尾時，開頭的資訊早已被稀釋，甚至「忘記」。按播放，觀察它如何閱讀：

模型從左讀到右。距離越遠的字，關係越難保留——句尾的「牠」很難連回句首的「小狗」。

此外，因為必須照順序一個一個處理，前一個算完才能算下一個， 所以無法平行運算，訓練起來很慢。這兩個瓶頸——遠距離會忘、無法平行——正是下一個方法要解決的。

02

注意力機制：讓每個字直接看向全句

注意力的想法很直接：不再照順序慢慢傳遞，而是讓每個字直接看向句中所有字， 再自行決定「該關注誰、關注多少」。點一個字，看它把注意力放在哪裡（顏色越亮＝關注越多）：

點上方任一個字，觀察它的注意力分布。試試代名詞「牠」。

看「牠」——它幾乎把注意力全放在句首的小狗。距離不再是問題： 再遠的關係，注意力都能一步直達。而且每個字的注意力是同時算的， 可以平行處理。兩個瓶頸，一次解決。

03

它怎麼決定「該關注誰」？Q · K · V

機制類似查資料。句中每個字，都同時帶著三種角色：

Q

Query　問題

我正在找什麼樣的資訊？

K

Key　索引

我是什麼、能不能被找到？

V

Value　內容

被找到後，我提供的資訊。

某個字的 Query 會去和所有字的 Key 比對， 越相符，注意力越高，就多取一點那個字的 Value。以「牠」為例，它的 Query 去比對各字的 Key：

「牠」的 Query　→　比對各字的 Key（相符程度）
結果：「小狗」的 Key 最相符 → 注意力集中於它 → 主要取用「小狗」的 Value。這就是「牠 ＝ 小狗」的由來。

但注意力不管「順序」——所以要補上位置

注意力只看「誰和誰相符」，本身並不知道字的先後。可是順序會改變意思。切換看看：

我 打 你
主詞＝我，受詞＝你　→　被打的是「你」

同樣三個字，順序一換，意思相反。因此 Transformer 會替每個字額外加上一個 位置編碼 positional encoding，讓模型同時知道「是什麼字」和「排在第幾個」。

04

Transformer 與「Attention is all you need」

把「注意力層」一層層堆疊起來（每層後面接一個前饋網路），再加上位置編碼， 就是 Transformer 的骨架：

資料由下往上：文字 → 轉成向量並標上位置 → 反覆經過「注意力＋前饋」N 層 → 輸出。

而 2017 年那篇論文的標題，就是它的爆點——「Attention is all you need」：

以前（RNN）

  • 必須「循序」一個字一個字算
  • 遠距離的關係容易遺忘
  • 無法平行 → 訓練慢、難做大

這篇之後（Transformer）

  • 拿掉循序結構，只用注意力
  • 再遠的關係都能一步直達
  • 全部字可平行運算 → 又快又能無限做大

「拿掉循序、只留注意力」讓模型可以平行、放大。正因如此，後來才能訓練出參數量巨大的 GPT——也就是 ChatGPT 的底層。今天幾乎所有大型語言模型，都是 Transformer。

回顧

本課重點

  • 01舊方法（RNN）循序閱讀，遠距離會忘、又無法平行。
  • 02注意力＝每個字直接看向全句、自行決定關注誰，一步直達且可平行。
  • 03用 Q·K·V 決定關注度（Query 比對 Key，取用 Value）；另加位置編碼補上順序。
  • 04堆疊注意力層即 Transformer；「只用注意力」帶來平行與放大，催生了 GPT。
